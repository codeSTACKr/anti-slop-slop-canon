# Onboarding

Onboarding behavior is not present in Phase 1. The approved contract requires an adaptive, one-question-at-a-time flow with personalize, defaults-only, and defer choices. It must preview a complete profile plus written and spoken examples before saving anything.

Supplied samples and analysis are not retained by default. Implementation belongs to Phase 4; the Phase 1 onboarding fixture records the observable behavior it must satisfy.
